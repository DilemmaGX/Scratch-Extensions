Animator Extension Document

IMPORTANT

This document is based on Animator v1.0.1, thus it might be different from the current version until a grand update. The actual extension shall govern.

Introduction

The initial intention behind the Animator extension was to assist developers in creating smooth animations based on time. The first version of Animator was a small collection of functions created using the original Scratch, which was not based on time. Instead, the first version of Animator used a rough and unstable method of animation completion determination: rounding the coordinates and checking for equality. The second version of Animator, which utilized a small number of Gandi-IDE extensions, implemented a time-based animation completion determination method. However, due to limitations in the original Scratch, a large number of easing functions could not be pre-set. As a result, the second version of Animator quickly stopped receiving updates after a major iteration. I then shifted my focus towards developing a more powerful, versatile, and precise Animator extension, which yielded successful results.

The development cycle for Animator.js was relatively short, starting from September and continuing until November. Many community extension developers also participated in the development and testing of Animator.js. The list of key individuals involved in the development and testing process is as follows (in no particular order): DilemmaGX, Kuke MC, FurryR, YUEN, and wit_cat.

Feature Overview

Animator is divided into two main parts: easing calculations and effect parameters. The easing calculations part is inspired by ZTianming's blog (https://www.cnblogs.com/ZTianming/p/15390536.html) and is responsible for the execution of animations. The effect parameters part is similar to the effect retrieval feature in the Arkos extension, but it is more concise (included here for convenience of use). This feature is an advanced functionality used for customizing the usage scenarios of easing calculations.

Easing functions

The first part, which is an advanced functionality, primarily involves instantaneous value calculation and serves as the fundamental basis. The second part is similar to the original "slide to x,y within n seconds" feature and is relatively commonly used. However, please note that, except for the first part, the remaining three parts are not executed instantaneously, meaning they cannot be interrupted. If there is a need for interruptible animations, customization using the first part is required.

Effect Parameters

To obtain instantaneous effect values, it is recommended to use it in conjunction with the first easing calculation. This feature is essentially equivalent to the effect retrieval in the Arkos' Extensions (added here for ease of loading and retrieval).
